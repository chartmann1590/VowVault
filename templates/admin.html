How It Works

Email Upload Process:
  1. Users email photos to the monitor address
  2. System checks for photo attachments
  3. Photos are automatically added to gallery
  4. Confirmation email sent to user
  5. Non-photo emails get rejection notice
Accepted Formats:
  • JPG / JPEG
  • PNG
  • GIF
  • WebP
Setup Notes:
  • Gmail requires App Passwords
  • Enable "Less secure app access" for some providers
  • Check your email provider's IMAP settings

SSO Configuration

When enabled, admin access will require SSO login instead of just the admin key
Must match the redirect URI configured in your OAuth provider
Custom OAuth URLs (for custom provider)
Access Control
Users with emails from these domains will be granted admin access
Specific email addresses that will be granted admin access
When enabled, the admin key will still work as a backup authentication method

How It Works

SSO Authentication Process:
  1. User clicks "Sign in with SSO" on admin page
  2. Redirected to OAuth provider (Google, Azure, etc.)
  3. User authenticates with their credentials
  4. Provider returns user information
  5. System checks email against allowed list
  6. If authorized, user gains admin access
Supported Providers:
  • Google OAuth 2.0
  • Microsoft Azure AD
  • Okta
  • Custom OAuth 2.0
Setup Notes:
  • Configure OAuth app in your provider
  • Set redirect URI to /sso/callback
  • Add authorized domains/emails
  • Test with a small group first
  • Keep admin key as fallback initially
